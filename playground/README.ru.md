# Playground для confluence.js

Самодостаточные мини-проекты, чтобы попробовать `confluence.js` на живом Confluence: заполняете небольшой конфиг и сразу получаете настоящий ответ.

Каждая папка — **самостоятельный npm-проект** со своим `package.json`, своими зависимостями и своей инструкцией. Они запускаются отдельно, и любой можно целиком перенести в другой репозиторий.

## Сценарии

| Сценарий | Что показывает |
|---|---|
| [`oauth2/`](./oauth2/) | Весь поток OAuth 2.0 (3LO): согласие в браузере → токены → cloud id → запросы, с обновлением ротируемого токена, которое видно в выводе |

## Перед запуском

Сценарии линкуются на **локальную** сборку (`"confluence.js": "file:../.."`), поэтому видят ещё не опубликованные изменения. Соберите корень один раз:

```bash
# из корня репозитория
pnpm install
pnpm build
```

Дальше заходите в нужный сценарий и следуйте его `README.md`.

## Как добавить сценарий

1. Скопируйте существующую папку в `playground/<имя>/`.
2. Замените `src/`, поправьте `name` в `package.json`, перепишите `README.md`.
3. Добавьте строку в таблицу выше.

Общих файлов между сценариями нет — намеренно: так каждый остаётся независимым, и его можно унести к себе, ничего предварительно не распутывая.

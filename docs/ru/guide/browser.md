# Браузеры

Пакет работает в браузере без изменений. Сборка одна, а не отдельные для Node и для веба: код ветвится по возможностям среды, а не по тому, что это за среда.

## Прочитайте до того, как выкатите

Обращение к Confluence прямо со страницы означает, что учётные данные лежат в браузере.

API-токен во фронтенде виден любому, кто откроет devtools, и несёт полные права выпустившей его учётной записи. Client secret от OAuth 2.0 ещё хуже — он вдобавок позволяет выпускать токены. Ни тому, ни другому не место в бандле.

К тому же Confluence Cloud не отдаёт CORS-заголовки для сторонних источников, так что запрос с вашего домена браузер отклонит ещё до того, как тот дойдёт до Atlassian.

Рабочих схем остаётся две:

- **Сервер посередине.** Страница ходит в ваш бэкенд, бэкенд держит учётные данные и ходит в Confluence. Для почти любого продукта это и есть верный ответ.
- **Первосторонняя площадка.** Приложение Forge или Connect, либо расширение, где платформа сама даёт и источник, и учётные данные.

Браузерная поддержка существует ради этих площадок — а также инструментов, песочниц, страниц документации и тестов, — чтобы все они пользовались одной библиотекой. Это не приглашение положить токен в веб-страницу.

## Импорт с CDN

Любой CDN, разрешающий зависимости, отдаёт пакет как он опубликован:

```html
<script type="module">
  import { createV2Client } from 'https://esm.sh/confluence.js';
</script>
```

jsDelivr делает то же через эндпоинт `+esm`:

```js
import { createV2Client } from 'https://cdn.jsdelivr.net/npm/confluence.js/+esm';
```

Опубликованная точка входа импортирует `zod` голым спецификатором, а его браузер сам разрешить не может — перечисленные CDN переписывают его перед отдачей. Хостинг, отдающий файлы как есть, на этом импорте упадёт.

Для таких случаев и для простого `<script type="module">` без всякого резолвера есть самодостаточная сборка с уже вмонтированным zod:

```js
import { createV2Client } from 'https://cdn.jsdelivr.net/npm/confluence.js/dist/browser.js';
```

Это один файл примерно на 45 kB в gzip. Когда есть возможность, берите варианты выше: они оставляют бандлеру свободу переиспользовать один экземпляр zod и выбросить всё, что вы не вызываете.

## Аттачменты

Содержимое, уже находящееся в памяти — `File` из `<input>`, `Blob`, `Uint8Array`, строка, — уходит через `FormData`. `File` браузер читает с диска сам, поэтому большая загрузка не оседает в памяти:

```ts
const [file] = input.files;

await confluence.contentAttachments.createAttachment({
  id: pageId,
  attachments: { filename: file.name, content: file },
});
```

Поведение расходится там, где библиотеке передают `ReadableStream`. Отправка тела запроса потоком [не является baseline](https://developer.mozilla.org/en-US/docs/Web/API/Request/duplex): Chromium умеет по HTTP/2, Firefox и Safari не умеют вовсе. Поэтому:

- где стриминг работает — поток уходит потоком;
- где нет — поток сначала вычитывается в `Blob` и отправляется так.

Откат стоит памяти, поэтому он именно откат, а не поведение по умолчанию. Если прокси опустит соединение до HTTP/1.1, Chromium отклонит загрузку — а поток к тому моменту уже частично прочитан, и молча повторить запрос библиотека не может. Ошибка выходит наружу; если столкнулись — передавайте `Blob`.

Родные типы Node продолжают работать везде. `Buffer` — это `Uint8Array`, а `Readable` — `AsyncIterable`, так что оба принимаются; просто они не названы в типе, и именно поэтому декларации компилируются в проекте без `@types/node`.

## Что проверяется

На каждое изменение в CI работают две проверки:

- собранный пакет сканируется на всё node-специфичное — импорт `node:`, обращение к `process` или `Buffer`, а также утечку типа Node в публичные декларации, которая сломала бы `tsc` браузерным потребителям;
- собранный пакет загружается в headless Chromium в обоих видах поставки, делает запрос, загружает аттачмент и получает ошибку, которую сужает предикатом.

## Сужение ошибок между бандлами

Пользуйтесь [предикатами](./error-handling), а не `instanceof`. Они проверяют брендированный символ, а не цепочку прототипов, поэтому продолжают работать при разбиении на чанки, при минификации, переименовывающей классы, и когда на странице оказались две копии пакета:

```ts
import { isNotFoundError } from 'confluence.js';

if (isNotFoundError(error)) return null;
```
